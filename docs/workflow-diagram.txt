================================================================================
                        PHASE-BY-PHASE PIPELINE
================================================================================


  PHASE 0: DISCOVERY
  ===================
  Mode: Plan Mode (read-only, user Q&A)
  Agent: orchestrator (no subagent dispatch)

  User answers 8+ questions:
    Q1: Model path or description
    Q2: Primary metric (accuracy, loss, PSNR, F1, etc.)
    Q3: Lower is better? (for primary metric)
    Q4: Target value (optional)
    Q5: Training command
    Q6: Eval command (if separate)
    Q7: Divergence metric (default: loss)
    Q7a: Divergence lower_is_better? (default: True)
    Q8: Data paths (train, val)
    Q9: Environment manager (conda/venv/none)
    Q10: Model category (supervised/RL/generative)
    Q11: Budget mode (standard/extended/autonomous)
    Q12: User papers (optional, for research)

  +-- RL polarity validation -------------------------------------------+
  | If divergence_metric contains "reward"/"return"/"score"             |
  | AND divergence_lower_is_better = True (default):                    |
  |   WARN: "For reward-like metrics, divergence means dropping —       |
  |          confirming lower_is_better=False?"                         |
  +---------------------------------------------------------------------+

  Saves: pipeline-state.json (user_choices), user-choices-backup.json

                    |
                    v

  PHASE 1: UNDERSTAND MODEL
  ==========================
  Agent: orchestrator reads code directly
  External: claude-mem:mem-search (recall past projects)

    - Read model files, config files
    - python3 scripts/gpu_check.py  -->  GPU count, VRAM
    - Detect framework (PyTorch/TF/JAX/sklearn/XGBoost/LightGBM)
    - Determine model_category if not set

  Writes: dev_notes.md (session log starts)

                    |
                    v

  PHASE 2: PREREQUISITES
  =======================
  Agent: prerequisites-agent --> ml-optimizer:prerequisites skill

    +--------------------------------------------------------------+
    |  Step 1: Detect environment (conda/venv/system)              |
    |  Step 2: Scan imports, check missing packages                |
    |  Step 3: Detect dataset format (CSV/ImageFolder/HuggingFace) |
    |  Step 4: Validate dataset paths and structure                |
    |  Step 5: Install missing dependencies                        |
    |  Step 6: Prepare data if needed (preprocessing)              |
    |  Step 6.1: Dry-run validation (NEW)                          |
    |    timeout 120 <train_command_with_minimal_steps>            |
    |  Step 7: Write prerequisites.json report                     |
    +--------------------------------------------------------------+

  Reads: user_choices (train_command, data paths, env_manager)
  Writes: experiments/results/prerequisites.json
          experiments/prepared-data/ (if preprocessing needed)

                    |
                    v

  PHASE 3: BASELINE
  ==================
  Agent: experiment-agent --> ml-optimizer:baseline skill

    +--------------------------------------------------------------+
    |  1. Run training with default HPs                            |
    |  2. Parse logs: python3 scripts/parse_logs.py <logfile>      |
    |  3. Check divergence: python3 scripts/detect_divergence.py   |
    |  4. Profile GPU memory usage (skip for tabular ML)           |
    |  5. Estimate throughput (steps/sec, time/epoch)              |
    |  6. Write baseline.json                                      |
    +--------------------------------------------------------------+

  Reads: prerequisites.json, user_choices
  Writes: experiments/results/baseline.json
          experiments/logs/baseline/train.log

                    |
                    v

  PHASE 4: USER CHECKPOINT
  =========================
  Agent: orchestrator (user interaction)

  Presents options to user:
    +--------------------------------------------------------------+
    | 1. Continue to research (web search for techniques)          |
    | 2. Skip research, go directly to HP tuning                   |
    | 3. Adjust baseline config and re-run                         |
    | 4. Stop here                                                 |
    | 5. Generate method proposals (research with source: "both")  |
    +--------------------------------------------------------------+

  Option 5 triggers:
    - Ask user for method_proposal_scope (training/architecture/full)
    - research-agent --> ml-optimizer:research (source: "both")
    - User confirms proposals
    - implement-agent --> ml-optimizer:implement
    - Creates branches/backups for each proposal

                    |
     (option 1 or 5 chosen)
                    v

  PHASE 5: RESEARCH
  ==================
  Agent: research-agent --> ml-optimizer:research skill

    +--------------------------------------------------------------+
    |  Source modes:                                               |
    |    "web"       — WebSearch + WebFetch only                   |
    |    "knowledge" — LLM training knowledge (confidence cap 7/10)|
    |    "both"      — LLM proposals + web validation              |
    |                                                              |
    |  Scope levels:                                               |
    |    "training"      — HPs, schedulers, augmentation (safest)  |
    |    "architecture"  — model changes, loss functions           |
    |    "full"          — any change including data pipeline      |
    |                                                              |
    |  Fuzzy deduplication (NEW):                                  |
    |    - Normalize: lowercase, strip suffixes                    |
    |    - Substring containment check                             |
    |    - Common abbreviation matching (lr↔learning rate, etc.)   |
    |    - >70% word overlap → treat as duplicate                  |
    |                                                              |
    |  Priority scoring:                                           |
    |    score = (impact × confidence) / (11 - feasibility)        |
    +--------------------------------------------------------------+

  Reads: baseline.json, user_choices, existing findings files
  Writes: experiments/reports/research-findings.md
          experiments/reports/research-findings-method-proposals.md (if "both")

                    |
                    v

  PHASE 6: IMPLEMENT
  ===================
  Agent: implement-agent --> ml-optimizer:implement skill
  Post-review: feature-dev:code-reviewer (quality check)

    +---------------------------------------------------------------+
    |  Strategy A — Git Branch (default):                           |
    |    Parallel via git worktrees (one Agent per proposal):       |
    |      1. git checkout -b ml-opt/<slug> from base branch        |
    |      2. Apply code changes                                    |
    |      3. Validate syntax (py_compile)                          |
    |      4. Run dry-run test                                      |
    |      5. Record in implementation-manifest.json                |
    |                                                               |
    |  Strategy B — File Backup (non-git fallback):                 |
    |    Restore-before-apply pattern (prevents cross-contamination)|
    |      1. Create baseline backup (_baseline/)                   |
    |      2. For EACH proposal:                                    |
    |         a. Restore ALL files from _baseline/ (clean slate)    |
    |         b. Apply this proposal's changes                      |
    |         c. Validate syntax                                    |
    |         d. If pass: backup to backups/<slug>/                 |
    |         e. Restore from _baseline/ (clean for next)           |
    |      3. Experiments run SEQUENTIALLY (not parallel)           |
    +---------------------------------------------------------------+

  Reads: research-findings.md, user-confirmed proposal indices
  Writes: experiments/results/implementation-manifest.json
          Git branches: ml-opt/<slug> (one per proposal)

                    |
                    v

  PHASE 7: EXPERIMENT LOOP
  =========================
  Budget modes:
    auto:       max_experiments = max(num_gpus, 1) × difficulty_multiplier
                  easy=×8, moderate=×15, hard=×25 (agent judges task difficulty)
    autonomous: unlimited (runs until 3 consecutive stop recommendations)
    custom:     user-specified experiment limit

  +======================================================================+
  |                                                                      |
  |  ITERATION START                                                     |
  |  +-----------------------------------------------------------------+ |
  |  |  7.1  HP-TUNE                                                   | |
  |  |  Agent: tuning-agent --> ml-optimizer:hp-tune skill             | |
  |  |                                                                 | |
  |  |  Inputs: past results (grouped by code_branch), remaining_budget| |
  |  |  Iter 1: one config per branch + one baseline HP config         | |
  |  |  Iter 2+: prune underperforming branches, focus configs         | |
  |  |  Cap: min(max(num_gpus, 1), remaining_budget) proposals         | |
  |  |                                                                 | |
  |  |  WARNING: If branches dropped due to budget, log to             | |
  |  |  error_tracker with priority order (impact × confidence)        | |
  |  +-----------------------------------------------------------------+ |
  |                    |                                                 |
  |                    v                                                 |
  |  +-----------------------------------------------------------------+ |
  |  |  7.2  EXPERIMENT                                                | |
  |  |  Agent: experiment-agent --> ml-optimizer:experiment skill      | |
  |  |                                                                 | |
  |  |  - Generate training scripts per config                         | |
  |  |  - Run training (parallel across GPUs if available)             | |
  |  |  - Parse logs, collect metrics                                  | |
  |  |  - Write exp-NNN.json results                                   | |
  |  |  - Timeout: baseline_time × 3 (iter 1), × 2 (iter 2+)           | |
  |  +-----------------------------------------------------------------+ |
  |                    |                                                 |
  |                    v                                                 |
  |  +-----------------------------------------------------------------+ |
  |  |  7.3  MONITOR (concurrent with experiment)                      | |
  |  |  Skill: ml-optimizer:monitor                                    | |
  |  |  (skipped for tabular ML: sklearn, XGBoost, LightGBM)           | |
  |  |                                                                 | |
  |  |  - Watch divergence_metric (default: "loss")                    | |
  |  |  - python3 scripts/detect_divergence.py '<values>'              | |
  |  |  - Returns: healthy | diverged | unmonitored                    | |
  |  |                                                                 | |
  |  |  "unmonitored": metric not found after fallbacks                | |
  |  |    → warn user, rely on hard timeout as safety net              | |
  |  |  "diverged": kill training, mark experiment as diverged         | |
  |  |                                                                 | |
  |  |  Recovery mechanisms:                                           | |
  |  |    - Early batch abort: ≥2 diverge within 60s → cancel rest     | |
  |  |    - All-diverge recovery: all diverge → retry halved LR        | |
  |  |    - OOM feedback: record max_batch_size for next hp-tune       | |
  |  +-----------------------------------------------------------------+ |
  |                    |                                                 |
  |                    v                                                 |
  |  +-----------------------------------------------------------------+ |
  |  |  7.4  ANALYZE + SPECULATIVE HP-TUNE (parallel)                   | |
  |  |  Agent: tuning-agent --> ml-optimizer:analyze skill             | |
  |  |                                                                 | |
  |  |  Runs IN PARALLEL with a speculative hp-tune call:              | |
  |  |    - If analyze says "continue" and speculative proposals pass  | |
  |  |      validation → use them immediately (saves 30-60s idle time) | |
  |  |    - If analyze says stop/pivot → discard speculative proposals | |
  |  |                                                                 | |
  |  |  Inputs: results_dir, primary_metric, lower_is_better,          | |
  |  |          remaining_budget                                       | |
  |  |                                                                 | |
  |  |  Decision tree:                                                 | |
  |  |    "continue"         → next HP iteration                       | |
  |  |    "stop"             → go to Phase 9 report                    | |
  |  |    "method_proposal"  → trigger 7.5 research pivot              | |
  |  |      (requires remaining_budget >= 3)                           | |
  |  |    "qualitative_change" → trigger full research                 | |
  |  |      (requires remaining_budget >= 5)                           | |
  |  +-----------------------------------------------------------------+ |
  |                   |                                                  |
  |         +---------+--- ------+----------+                            |
  |         |         |          |          |                            |
  |    "continue"  "stop"  "method_   "qualitative_                      |
  |         |         |    proposal"   change"                           |
  |         |         |          |          |                            |
  |         |         v          +----+-----+                            |
  |         |    Phase 9              |                                  |
  |         |                         v                                  |
  |  +------+----------------------------------------------------------+ |
  |  |  7.5  RESEARCH PIVOT (triggered by analyze)                     | |
  |  |  Agent: research-agent --> ml-optimizer:research (source:"both")| |
  |  |  Then: implement-agent --> ml-optimizer:implement               | |
  |  |  User confirmation required for scope + proposals               | |
  |  +-----------------------------------------------------------------+ |
  |         |                                                            |
  |         v                                                            |
  |  +----------------------------------------------------------------+  |
  |  |  7.6  AUTONOMOUS RESEARCH CADENCE                              |  |
  |  |  (autonomous mode only, method_proposal_scope set)             |  |
  |  |                                                                |  |
  |  |  Every hp_batches_per_round batches (default 3):               |  |
  |  |    Auto-trigger research → implement cycle                     |  |
  |  |    NO user confirmation needed                                 |  |
  |  |    If no new proposals: double cadence (exponential backoff)   |  |
  |  +----------------------------------------------------------------+  |
  |         |                                                            |
  |         v                                                            |
  |  +----------------------------------------------------------------+  |
  |  |  7.7  MID-PIPELINE REVIEW (auto-triggered)                     |  |
  |  |  Skill: ml-optimizer:review (scope: "session")                 |  |
  |  |                                                                |  |
  |  |  Triggers:                                                     |  |
  |  |    - wasted_budget pattern ≥ 3 occurrences                     |  |
  |  |    - 2 consecutive batches with zero successful experiments    |  |
  |  |                                                                |  |
  |  |  Output: course correction suggestions in session-review.md    |  |
  |  +----------------------------------------------------------------+  |
  |         |                                                            |
  |         +-- Loop back to 7.1 HP-TUNE (if budget remains) -->             |
  |                                                                      |
  +============================ END LOOP ================================+

                    |
      (budget exhausted, or "stop" recommendation,
       or 3 consecutive stops in autonomous mode)
                    v

  PHASE 8: METHOD STACKING (post-loop, if applicable)
  ====================================================
  Agent: orchestrator dispatches implement-agent + experiment-agent
  Requires: git branch strategy (skipped for file_backup projects)

  Trigger: experiment loop ends AND >= 5 methods improved over baseline

    +--------------------------------------------------------------+
    |  1. Rank methods by improvement (descending) using           |
    |     rank_methods_for_stacking() from result_analyzer.py      |
    |                                                              |
    |  2. Initialize: best method branch -> ml-opt/stack-1         |
    |                                                              |
    |  3. For each remaining method (in rank order):               |
    |     a. Create ml-opt/stack-<N> by merging next method        |
    |     b. Clean merge -> proceed; conflicts -> implement-agent  |
    |     c. Run experiment with best individual HPs               |
    |     d. If improved: keep stack, optional HP-tune (1-2 iters) |
    |     e. If degraded: skip method, re-branch from last success |
    |                                                              |
    |  4. Save stacking state to pipeline-state.json (resumable)   |
    +--------------------------------------------------------------+

  Reads: results/exp-*.json, implementation-manifest.json, baseline.json
  Writes: results/exp-*.json (stacked_default_hp / stacked_tuned_hp)
          Git branches: ml-opt/stack-<N>

                    |
      (stacking complete or skipped)
                    v

  PHASE 9: REPORT
  ================
  Agent: tuning-agent --> ml-optimizer:report skill

    +--------------------------------------------------------------+
    |  Compile final optimization report:                          |
    |    - Best experiment config and metrics                      |
    |    - Ranking of all experiments                              |
    |    - HP sensitivity analysis (Spearman correlations)         |
    |    - ASCII charts (comparison, timeline, sensitivity)        |
    |    - Matplotlib progress chart                               |
    |    - Error pattern summary                                   |
    |    - Proposal outcome tracking                               |
    |                                                              |
    |  Three-tier comparison (when method_tier present):           |
    |    Tier 1: baseline                                          |
    |    Tier 2: method_default_hp (isolated method effect)        |
    |    Tier 3: method_tuned_hp (combined HP + method effect)     |
    |                                                              |
    |  Stacking comparison (when stacking results present):        |
    |    stacked_default_hp (combined methods, individual HPs)     |
    |    stacked_tuned_hp (combined methods, tuned HPs)            |
    +--------------------------------------------------------------+

  Reads: All experiments/results/*.json, error-log.json
  Writes: experiments/reports/optimization-report.md
          experiments/reports/progress_chart.png

                    |
                    v

  OPTIONAL: END-OF-SESSION REVIEW
  ================================
  Skill: ml-optimizer:review (scope: "session")

    - Self-improvement analysis
    - What worked, what didn't
    - Suggestions for future sessions

  Writes: experiments/reports/session-review.md

================================================================================
                              LEGEND
================================================================================

  Agent dispatch:    orchestrator ---> agent ---> skill
  User interaction:  [User]  <--->  [Orchestrator]
  Conditional:       (condition) ---> branch
  External skill:    context7, claude-mem, feature-dev, superpowers
  Data flow:         Reads: <file>  |  Writes: <file>

  Agents with "ultrathink": Extended reasoning for complex decisions
  Agents "procedural": Direct execution, no extended reasoning

  Three-tier + stacking fields in exp-*.json:
    "method_tier":     "baseline" | "method_default_hp" | "method_tuned_hp"
                       | "stacked_default_hp" | "stacked_tuned_hp"
    "proposal_source": "paper" | "llm_knowledge" | null
    "code_branches":   ["ml-opt/a", "ml-opt/b"] (stacking only)
    "stacking_order":  integer position in stacking sequence
    "stack_base_exp":  exp_id of previous stack step
